# AI 编辑器 413 Content Too Large 问题

**日期：** 2026-04-10  
**问题：** AI 编辑器生成图片时报 413 (Content Too Large) 错误  
**影响功能：** AI 图片生成

---

## 问题现象

点击 AI 编辑器的生成按钮后，返回错误：
```
POST https://imagetoolss.com/api/generate 413 (Content Too Large)
```

---

## 问题根因

Vercel 默认对请求体大小有限制（1MB）。

AI 编辑器在发送请求时，将参考图片以 **base64 格式**包含在 JSON 请求体中：
```javascript
body: JSON.stringify({
  prompt: prompt.trim(),
  referenceImages: referenceImages, // base64 格式的数组
})
```

base64 编码会使数据增大约 33%，多张或大尺寸图片很容易超过 1MB 限制。

---

## 解决方案

### 采用 Supabase Storage 方案

**原理：** 前端将图片上传到 Supabase Storage，获得公开 URL，再将 URL 发送给 API。

**流程：**
```
1. 用户选择参考图片
2. 前端通过 /api/upload 上传到 Supabase Storage
3. API 返回公开 URL
4. /api/generate 接收 URL，从 URL 下载图片并转为 base64
5. 调用 OpenRouter API 生成图片
```

**优点：**
- 避免 base64 大小膨胀
- 请求体保持在 1MB 以内
- 复用 Supabase Storage 服务

## 代码修改

### `/api/generate/route.ts`

修改前：
```typescript
const { prompt, referenceImages = [] } = body
// OpenRouter 请求直接包含 base64
```

修改后：
```typescript
const { prompt, referenceImageUrls = [] } = body
// 先从 URL 下载图片，再转 base64
if (referenceImageUrls?.length > 0) {
  const imageData = await Promise.all(
    referenceImageUrls.slice(0, 2).map(async (url) => {
      const imgResponse = await fetch(url)
      const buffer = await imgResponse.arrayBuffer()
      const base64 = Buffer.from(buffer).toString("base64")
      return `data:${mimeType};base64,${base64}`
    })
  )
}
```

### 前端 `ai-editor.tsx`

修改 `handleGenerate` 函数：

```typescript
// 先上传图片获取 URL
if (referenceFiles.length > 0) {
  const uploadPromises = referenceFiles.map(async (file) => {
    const formData = new FormData()
    formData.append("file", file)
    const response = await fetch("/api/upload", { method: "POST", body: formData })
    const data = await response.json()
    return data.url
  })
  referenceImageUrls = await Promise.all(uploadPromises)
}

// 用 URL 而不是 base64 调用生成 API
body: JSON.stringify({
  prompt: prompt.trim(),
  referenceImageUrls: referenceImageUrls,
})
```

## 经验教训

1. **base64 会使数据增大 33%**，传输大图片时要注意
2. **Vercel 请求体限制 1MB**，超出返回 413
3. **图片上传到云存储再传 URL**，比直接传 base64 更高效
